# 의약품 글쓰기 에이전트

> `data/articles/*.ts` 의약품 spoke 글을 작성할 때 이 파일을 읽는다.
> hooks(pre-edit-verify, q1q4-inject, track-source-read)가 자동 차단하는 항목은 여기서 다루지 않는다.
> 이 에이전트는 hooks가 **못 잡는 질적 판단**에 집중한다.

---

## 1. 콘텐츠 차별화 원칙 (복붙 금지)

같은 카테고리 내 수천 개 제품이 동일한 API 필드를 공유한다.
구조는 템플릿을 따르되, **내용은 이 제품만의 특성에서 출발**해야 한다.

### 작성 전 필수: 차별화 포인트 3가지 추출

글 작성 전 아래 항목을 소스 JSON에서 확인하고 메모한다.

| 항목 | 확인 내용 |
|------|----------|
| 주성분/함량 | 아세트아미노펜 500mg vs 325mg, 이부프로펜 200mg vs 400mg 등 |
| 제형 | 정제 / 캡슐 / 시럽 / 연고 / 액제 / 패치 / 좌제 |
| 복합 성분 | 단일 vs 복합(해열+항히스타민+진해 등), 카페인 포함 여부 |
| 타겟 | 성인 / 소아 / 임산부 / 고령자 / 특정 증상(야간용, 주간용) |
| 투여 경로 | 경구 / 외용 / 점안 / 비강 / 좌약 |
| 처방 구분 | 일반의약품(OTC) / 전문의약품(ETC) |

### 서론(heroDescription) 작성 규칙

- **이 제품을 검색하게 된 구체적 상황**으로 시작한다.
- 주성분·제형·타겟 중 이 제품의 핵심 1가지를 첫 문장에 담는다.
- 타제품과 직접 비교하지 않되, 이 제품이 선택받는 이유를 자연스럽게 녹인다.

**나쁜 예** (복붙): "감기에 걸렸을 때 약국에서 자주 보는 약이…" → 다른 감기약 서론과 동일
**좋은 예** (차별화): "밤에 기침이 심해 잠을 못 잘 때 야간용 감기약을 찾게 되는데…" (상황+타겟 기반)

### 본문 섹션 작성 규칙

- **성분 섹션**: 단일 성분이면 그 성분의 작용 기전 중심, 복합이면 각 성분 역할 분담 설명.
- **효과 섹션**: 적응증이 넓으면 가장 대표적 증상부터, 좁으면 해당 증상을 깊게.
- **복용법/사용법**: 제형마다 실용 팁이 다르다. 정제→물과 함께, 시럽→계량컵, 연고→면봉/거즈, 패치→부착 부위 교체.
- **타제품 비교 문장**: 본문 내 자연스럽게 1회 이상 포함. ("시럽형과 달리", "정제 제형이라", "카페인이 포함된 제품과 달리" 등)

## 2. Q1~Q4 사고 프레임워크 (왜 하는가)

> hooks(q1q4-inject.js)가 Q1 주석 존재를 강제한다. 여기서는 "잘 쓰는 법"을 설명한다.

- **Q1**(독자 상황): heroDescription 첫 문장을 결정. "궁금해서"는 금지 → 구체적 상황 1줄.
- **Q2**(독자 행동): 어느 섹션을 가장 두껍게 쓸지 결정. H2 순서는 고정이지만 깊이는 Q2가 결정.
  - "비교해서 고른다" → 비교표 추가
  - "절차를 따라 복용한다" → 복용법 두껍게
  - "부작용을 확인한다" → 부작용+주의사항 두껍게
- **Q3**(필수 정보): 예외/주의사항 3가지 이상. 소스 JSON에서 추출.
- **Q4**(전달 형식): 표/텍스트/경고박스 선택.

## 3. hooks가 못 잡는 질적 체크리스트

글 작성 후 아래를 자체 점검한다:

| 항목 | 체크 |
|------|------|
| heroDescription이 독자 상황에서 출발하는가 | hooks는 패턴만 차단, 질적 판단은 못 함 |
| 타제품 비교 문장이 본문에 1회 이상 있는가 | hooks 미검사 |
| FAQ 3개 중 1개가 실패/예외 시나리오인가 | hooks 미검사 (개수만 체크) |
| 내부링크(같은 카테고리 다른 제품) 삽입 기회가 있는가 | hooks 미검사 |
| 각 섹션이 이 제품 고유 특성을 반영하는가 | hooks 미검사 (길이만 체크) |
